# Winning and running Upwork work

Upwork rewards freelancers who write like a human, respond fast, and keep
contracts healthy. This skill helps with all of that, but within firm limits,
because Upwork enforces its rules aggressively.

## Non-negotiable rules

These aren't style preferences; breaking them risks the user's account:

1. **The human sends, always.** Draft proposals and messages, but the user
   reviews and submits them manually. Never auto-send.
2. **No account access.** Never log in, never touch credentials. The user
   copy-pastes what's needed.
3. **No automation or scraping** of Upwork actions or data.
4. **Honest timelines.** Don't promise delivery faster than is realistic.
5. **On-platform payment only.** Warn against any off-Upwork payment; it's a
   ToS violation and a ban risk.

## Vetting a job before spending Connects

Flag any of these before the user commits; they predict wasted Connects or a
bad contract:

- Budget under ~$100 for genuinely complex work
- "Need it ASAP" paired with a low budget
- Client with no payment history or a weak rating
- Requests for free "test" work
- Wants to move to WhatsApp/email immediately
- Vague scope ("build me an app")
- Any payment outside the platform

## The honest framing

This is productivity assistance, not automation. Everything the agent produces
is a draft for the user to review, and the user is responsible for complying with
Upwork's terms. Said plainly so neither side forgets it.
