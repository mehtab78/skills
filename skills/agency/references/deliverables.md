# Deliverables workflow

Move from raw input to a reviewed, on-brief deliverable, with a human approving
before anything reaches the client.

## Input → draft

1. **Gather inputs:** the client brief and preferences (from their file),
   references and examples, brand guidelines, and relevant prior work.
2. **Draft it:** follow the agency template if one exists, include every
   required section, mark gaps clearly as `[NEEDS INPUT]`, and leave review notes
   as HTML comments.
3. **Self-review first:** does it match the brief, is it complete, and is the
   quality good enough for client eyes?

## By deliverable type

- **Reports/documents:** executive summary → details → recommendations; add
  visuals where data supports them; offer both a full and a one-page version.
- **Presentations:** one idea per slide, minimal text, presenter notes, and a
  narrative arc (problem → approach → solution → results).
- **Creative assets:** give a few variations to choose from, include usage specs
  (sizes, formats, colors), and note any fonts/stock used.
- **Code/technical:** a README with setup, environment requirements, a QA
  checklist, and actual testing before delivery.

## Review and revisions

Before the user reviews: check against the brief, run the type's QA checklist,
and flag weak spots yourself. After their feedback: apply changes, note patterns
worth improving, and update templates if relevant.

Track versions (v1 draft → v2 client feedback with the changes listed → v3 final)
and honor revision limits, state the number of rounds in the proposal, track
usage, and quote additional rounds once the limit's hit.

## One piece, many formats

When content needs adapting, say an article into social, produce each format
for its platform: the full article, a LinkedIn post (key insight + CTA), a
Twitter/X thread, an Instagram carousel, and a newsletter blurb, each matched to
that platform's length, tone, and visuals.

## Delivery checklist

Before sending: every brief requirement met, proofread/QA passed, correct formats
and file types, clear file names, a cover note drafted, next steps included.
After: confirm receipt, log the delivery, archive the final files in the client
folder, and update the case study if it's notable.
